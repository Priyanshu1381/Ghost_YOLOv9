# Ghost YOLO Notebook README

This README describes all changes made in `ghost-yolo (3).ipynb` to convert a YOLOv9-style pipeline into a Ghost YOLO training and inference workflow.

## What changed

### 1. Environment setup
- Installed `ultralytics` for YOLOv9 support.
- Installed `pandas` for better metric tables and presentation.

### 2. Custom metric tracking
- Added a callback function `on_fit_epoch_end(trainer)` to extract training and validation metrics at each epoch.
- Collected:
  - Training loss (`train/box_loss + train/cls_loss + train/dfl_loss`)
  - Validation loss (`val/box_loss + val/cls_loss + val/dfl_loss`)
  - `mAP@50` metric
  - Inference time per image
- Stored these metrics in a list for presentation and screenshot-friendly export.

### 3. Ghost YOLO architecture patching
- Imported `GhostConv`, `RepCSP`, and `RepNCSPELAN4` from `ultralytics.nn.modules`.
- Monkey-patched `RepNCSPELAN4.__init__` to replace the standard convolutional path with `GhostConv` operations.
- This patch creates a GhostNet-inspired block inside the YOLO backbone.

### 4. Custom model configuration file
- Defined `custom_yaml` inside the notebook for the Ghost YOLO architecture.
- Wrote the configuration to `yolov9c-ghost.yaml`.
- The architecture includes:
  - `GhostConv` layers at the first two backbone stages
  - `RepNCSPELAN4` blocks at deeper backbone stages
  - `ADown` downsampling blocks
  - `SPPELAN` at the final backbone output stage
  - YOLO-style multi-scale head with feature fusion and `Detect([nc])`

### 5. Model loading and training
- Loaded the model using `YOLO('yolov9c-ghost.yaml')`.
- Registered the custom callback with `model.add_callback('on_fit_epoch_end', on_fit_epoch_end)`.
- Training command:
  - `data='VOC.yaml'`
  - `epochs=50`
  - `imgsz=640`
  - `batch=32`
  - `device=[0, 1]`
  - `project='yolo_voc'`
  - `name='v9_train'`
  - `val=True`
  - `pretrained=False`

### 6. Results presentation and plotting
- Loaded `results.csv` from the YOLO run output.
- Reconstructed a clean presentation table with:
  - Epoch
  - Training Loss
  - Validation Loss
  - mAP@50 (Mean IoU)
- Used `matplotlib` and `seaborn` to plot training vs validation loss.
- Saved a high-resolution `.png` file: `custom_overfitting_analysis.png`.

### 7. Peak GPU memory reporting
- Calculated total peak GPU memory consumed using `torch.cuda.max_memory_allocated()`.
- Displayed the peak memory usage in GB.

### 8. Confusion matrix display
- Loaded YOLO-generated confusion matrix at `model.trainer.save_dir/confusion_matrix.png`.
- Displayed the image inline if available.

### 9. Video inference workflow
- Loaded the trained model from `/kaggle/working/runs/detect/yolo_voc/v9_train/weights/best.pt`.
- Set a Kaggle video path placeholder for sample video input.
- Used `trained_model.predict(source=video_path, save=True, stream=True)`.
- Collected and printed inference performance stats:
  - preprocess time
  - inference time
  - postprocess time
  - average time per frame
  - effective FPS
- Output is saved to `/kaggle/working/runs/detect/predict/`.

### 10. Robust results discovery
- Added logic to search for the newest `results.csv` in `/kaggle/working` and the local folder.
- This allows the notebook to reconstruct presentation tables even if the run folder name changes.

### 11. Presentation image retrieval
- Searched for the latest YOLO run `weights` folder and displayed available graphs:
  - `results.png`
  - `F1_curve.png`
  - `PR_curve.png`
  - `confusion_matrix_normalized.png`
  - `val_batch0_pred.jpg`

### 12. Tracking demo preparation
- Added a ByteTrack inference workflow at the end of the notebook.
- Loaded the trained model and configured video tracking with `tracker="bytetrack.yaml"`.
- This is a separate inference mode from plain `predict()` and produces tracked output on video.

## Files created or modified by the notebook
- `yolov9c-ghost.yaml` — Ghost YOLO model config generated by the notebook.
- `custom_overfitting_analysis.png` — saved training analysis figure.
- `yolo_voc/v9_train/results.csv` — YOLO training log output.
- `/kaggle/working/runs/detect/predict/` — directory for saved video inference results.

## Notes
- The notebook is written to run in a Kaggle-style environment with `/kaggle/working` and `/kaggle/input` paths.
- The Ghost YOLO changes are implemented by patching the existing Ultralytics blocks, not by editing the Ultralytics source files directly.
- `nc: 20` is configured for PASCAL VOC by default.

## Summary
This notebook converts standard YOLOv9 training into a Ghost YOLO variant by:
- patching backbone blocks to use `GhostConv`
- generating a `yolov9c-ghost.yaml` config file
- training the model from scratch on VOC
- collecting clean epoch-level metrics for presentation
- saving analysis plots and inference outputs
- supporting both video prediction and ByteTrack-based tracking
